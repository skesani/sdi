Introduction

Introduction#

The SDI API provides programmatic access to Synthetic Digital Immunity’s anomaly detection and autonomous security features.

Quick Start#

  1. Get your API key from your SDI configuration
  2. Make your first request using curl or one of our client libraries
  3. Handle responses including anomaly detection results

Base URL#

https://api.yourservice.com/api/sdi

For local development:

http://localhost:8080/api/sdi

Making Requests#

All requests should include:

  • Authorization header with your API key
  • Content-Type: application/json for POST requests
  • Proper HTTP method (GET, POST, etc.)

Example:

  -X POST \

  -H "Content-Type: application/json" \
  -d '{
    "method": "GET",
    "path": "/api/data"
  }'

Response Format#

All responses are JSON objects. Successful responses include a data field, while errors include an error field.

Rate Limits#

SDI API has rate limits to ensure fair usage:

  • Free tier: 1000 requests/hour
  • Pro tier: 10000 requests/hour
  • Enterprise: Custom limits

Rate limit headers are included in all responses:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1701234567

Versioning#

The SDI API is versioned. The current version is v1. Include the version in your requests:

/api/sdi/v1/analyze

We maintain backward compatibility within major versions. When we release a new major version, we’ll provide migration guides.
