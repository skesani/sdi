Analysis#

Analyze a request for anomalies and security threats.

POST /api/sdi/analyze#

Analyzes an HTTP request for anomalies and potential security threats.

Request#

 1

 6
 7
 8
 9

11
12

14

curl https://api.yourservice.com/api/sdi/analyze \
  -X POST \
  -H "Authorization: Bearer sdi_test_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "method": "GET",
    "path": "/api/users/123",
    "headers": {
      "User-Agent": "Mozilla/5.0",
      "Accept": "application/json"
    },
    "body": null,
    "serviceId": "user-service"
  }'

Parameters#

Parameter Type Required Description
method string Yes HTTP method (GET, POST, PUT, DELETE, etc.)
path string Yes Request path
headers object No HTTP headers as key-value pairs
body string No Request body content
serviceId string No Identifier for the service making the request

Response Fields#

Field Type Description
anomalyDetected boolean Whether an anomaly was detected
anomalyScore number Anomaly confidence score (0.0 to 1.0)
severity string Severity level: low, medium, high, critical
serviceId string Service identifier
timestamp number Unix timestamp of the analysis
pipelineTriggered boolean Whether the full PRE pipeline was triggered

Detection#

POST /api/sdi/detect#

Performs quick anomaly detection only, without triggering the full PRE pipeline.

  -d '{
